Monte Carlo Tree Search

Problem: Choosing the best move in a board game with millions of possible positions. Real-world: This is how AlphaGo defeated the world champion at Go. How MCTS solves it: It builds a decision tree by simulating thousands of random games from each possible move, tracking win rates per branch, and balancing exploration of new moves vs. exploitation of proven ones. Click the board to place a piece, or use Auto-run to watch it play against itself.

Problem: Nim — a pile of 12 stones. Each player takes 1–3 stones per turn; whoever takes the last stone loses. Real-world: Nim has a perfect mathematical strategy, making it a great test for whether MCTS converges to optimal play. How MCTS solves it: It simulates thousands of games from the current pile size, learning that keeping the pile at multiples of 4 is the winning strategy.

Problem: Connect 4 — drop discs into a 7×6 grid, first to get 4 in a row wins. Real-world: Connect 4 is a solved game but has over 4 trillion possible positions. Brute-force search is impossible; MCTS approximates optimal play. How MCTS solves it: It builds a search tree from the current board, simulating random games down each column to estimate the best drop position.

    for _ in range(n_simulations):
        node = root
 
        # 1. Selection: pick promising child
        while node.children:
            node = best_ucb(node)

        if node.visits > 0:
            node = node.expand()
 
        # 3. Simulation: random playout
        result = simulate(node)
 
        # 4. Backpropagation: update ancestors
        while node:
            node.visits += 1

Genetic Algorithm

Problem: Finding the optimal shape for an aircraft wing from millions of design parameters. Real-world: NASA and automotive companies use GAs to evolve aerodynamic designs that no human engineer would think of. How GA solves it: A population of candidate designs evolves over generations: the fittest survive, pairs crossover their traits, and random mutations introduce novelty, converging on the optimal design.

Problem: Assigning hundreds of flights to gates, crews, and time slots while minimizing delays and costs. Real-world: Airlines use evolutionary optimization for crew scheduling, a problem so complex that brute force would take longer than the age of the universe. How GA solves it: Each individual encodes a full schedule. Selection favors low-cost schedules, crossover combines good partial assignments from two parents, and mutation swaps random slots to escape local optima.

Problem: Allocating capital across hundreds of assets to maximize return while controlling risk. Real-world: Hedge funds and robo-advisors use evolutionary strategies to find portfolio weights that balance the Sharpe ratio across thousands of market scenarios. How GA solves it: Each individual is a vector of asset weights. Fitness is risk-adjusted return. Crossover blends two portfolios, mutation tweaks allocations, and the population converges on efficient-frontier solutions.

def genetic_algorithm(pop, generations):
    for gen in range(generations):
 
        # 1. Evaluate fitness
        scores = [fitness(ind) for ind in pop]
 
        # 2. Selection: tournament pick
        parents = tournament_select(pop, scores)
 
        # 3. Crossover: combine parents
        offspring = []
        for p1, p2 in pairs(parents):
            child = crossover(p1, p2)
            offspring.append(child)
 
        # 4. Mutation: random tweaks
        for ind in offspring:
            if random() < mutation_rate:
                mutate(ind)
 
        pop = offspring
 
    return best(pop)

def forward(x, weights):
    # Hidden layer: learn features
    h = x @ weights[0] + bias[0]
    h = relu(h)
 
    # Output layer: class scores
    out = h @ weights[1] + bias[1]
    return softmax(out)
 
def train(network, data, epochs):
    for epoch in range(epochs):
        for x, y in data:
            pred = forward(x, network)
            loss = cross_entropy(pred, y)
 
            # Backpropagation: compute gradients
            grads = backward(loss, network)
 
            # Update weights via SGD
            for w, g in zip(network, grads):
                w -= learning_rate * g
